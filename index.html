Editing videos with ffmpeg (for cowards)

because it can get a little scary out there

FFMPEG can be used to perform a lot of the simple operations that are generally carried out using popular NLE video editing applications (like Premiere Pro). This includes cropping, trimming, adding padding(black rectangles) to get the right aspect ratios, zooming in to videos/images, applying transition effects, adding audio, adjusting the volume, exporting into different formats, etc. These are mostly the parts I'll cover in this blog since that's mostly what I messed around with.

Prefacing something by saying it's complex is a bad idea.
Especially when it's not that hard.

It puts learners off (like me!) because they think "oh no it's probably too complex". I fiddled with MoviePy for a month before deciding it's way too slow. MoviePy is a fine abstraction, although slower because it loads each frame into memory before manipulating it. MoviePy also has its own host of issues and because of these bugs, is generally annoying to deal with. Although it was a pretty good introduction to video editing using code.

I looked for countless alternatives to ffmpeg (for example using scripting with Python on free video editing software like DaVinci Resolve) before finally coming back to it.

FFMPEG, on the other hand, is magic. Just like [h.264](https://sidbala.com/h-264-is-magic/).

How does ffmpeg edit videos? Question to explore

The difference between a simple filter graph and a complex filter graph is that the simple filter graph takes one input and has one output.
